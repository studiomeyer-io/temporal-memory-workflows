# Code of Conduct

## Our pledge

We want this repository to be a place where developers ship better durable workflows. That requires an environment where contributors of all backgrounds, skill levels, and time zones can show up, ask questions, file bug reports, and have technical disagreements without it turning personal.

## Our standards

Examples of behavior that build trust:

- Direct, technical, helpful communication.
- Acknowledging when a contribution is solid or when feedback is correct.
- Linking to documentation, Temporal SDK source, or a Workflow ID — not just opinions.
- Asking clarifying questions before assuming bad faith.
- Reviewing pull requests with concrete suggestions, not vague pushback.
- Citing the Iron Rules in the README when blocking a PR on determinism grounds, and explaining which rule and why.

Examples of behavior we will not accept:

- Insults, slurs, or harassment of any kind.
- Public personal attacks (technical disagreement is fine, personal attack is not).
- Sustained derailing of issues or pull requests.
- Publishing other contributors' private information without explicit permission.
- Sexualized language or imagery in any project space.

## Scope

This applies in any project space (GitHub issues, pull requests, discussions, commit messages, code comments) and in public when an individual is representing the project (conference talks, social media posts that link this repo, etc.).

## Enforcement

Project maintainers are responsible for clarifying these standards and may take corrective action — including warning, temporary ban, or permanent ban from contribution.

If you see behavior that violates this code, contact the maintainer at **matthias10121980meyer@gmail.com**. Reports will be reviewed promptly and confidentially.
